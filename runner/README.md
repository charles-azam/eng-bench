# Native-system benchmark runner

This directory contains the VPS runner for the corrected benchmark. It is deliberately gated: `run-one.sh` and schedule generation refuse to run until `/root/bench-v2/protocol/FROZEN` and a valid `manifest.sha256` exist.

## Isolation and fairness contract

- Every attempt gets a fresh `/root/bench-v2/runs/<run-id>-aNN` directory with a hash-recorded host input copy, writable workspace, private runtime state, full JSONL events, stderr, provider-egress log, timestamps, hashes, requested model, CLI version and machine-readable status.
- Bubblewrap clears the host environment before setting a minimal fixed environment. Only the selected system's credential is mounted. The other credential file remains an empty private-home placeholder, and the other provider host is absent from that run's proxy allowlist.
- The namespace sets `IS_SANDBOX=1`, a deliberate-sandbox marker recognized by the frozen Claude executable. This permits `--dangerously-skip-permissions` despite the host process retaining UID 0; the marker is truthful only because bubblewrap hides the host filesystem, clears the environment, unshares network/PID/UTS/IPC namespaces, and applies the fixed egress proxy described below.
- Before every attempt, `verify-protocol.sh` checks every recorded digest and exact file-set equality; an extra unlisted file fails closed instead of being copied into a workspace.
- The same external bubblewrap sandbox is used for Codex and Claude Code. Only `/workspace`, a private runtime home, system binaries, provider credentials and the runner are visible. `/root/bench`, `/root/eng-bench`, `/root/bench-v2/hidden` and other host paths are absent.
- The network namespace has no direct egress. A namespace-local TCP bridge reaches a host Unix socket, where `connect_proxy.py` permits HTTPS CONNECT only to `chatgpt.com` for Codex or `api.anthropic.com` for Claude Code. Every allowed or rejected hostname is logged without request content or credentials.
- Native permissions are bypassed only inside that external sandbox so both systems receive the same filesystem and network boundary.
- Claude loads `claude-settings.json`, where `switchModelsOnFlag` is false. The validator treats any fallback event or non-Fable assistant message as `model_contamination`. A fail-closed safeguard refusal is labeled `model_refusal`.
- Codex runs with user config and rules ignored, an ephemeral session, no web-search flag and GPT-5.6 Sol at `max` reasoning. Claude runs in safe mode with no persisted session, no slash commands, no MCP servers, no subagent tool, no Chrome and Fable 5 at `max` effort.
- The 60-minute limit sends `SIGINT`; `timeout` permits cleanup until the registered 75-minute hard limit, then kills the process. Timeouts are system failures and are never infrastructure-retried.

The credentials have to be readable by the CLI process and are therefore also technically readable by a shell child in the same sandbox. They are bind-mounted over empty placeholder files and never copied into artifacts. This is a credential-containment limitation of native agent CLIs, not a hidden-measurement leak; benchmark prompts are non-adversarial.

## Frozen protocol layout

Expected files:

```text
/root/bench-v2/protocol/
  FROZEN
  VERSION
  environment.json
  manifest.sha256
  matrix.tsv
  tasks/<task>/PROMPT.md
  tasks/<task>/<pack files...>
```

`VERSION` contains the frozen benchmark version on one line. `environment.json` is generated by `environment_manifest.py`; its SHA-256 becomes the evaluator's `runner_image_sha256`. It is a digest of the host package manifests, scientific Python stack, normalization virtual environment, critical CLI/sandbox binaries, proxy settings, and exact runner source file set—not an OCI image. Every attempt regenerates and byte-compares it before inference.

`manifest.sha256` contains paths relative to `protocol/` and excludes itself. `matrix.tsv` columns are:

```text
stage  task  system  first_replicate  last_replicate
```

The matrix should define separate stages such as `core-n3`, `core-extend-n5`, and `nstf-duty-ablation-n3`. Generate each deterministic randomized schedule with:

```bash
/root/bench-v2/runner/make-schedule.sh --stage core-n3 --seed 20260711
```

Review the generated schedule and hash, then run it sequentially with:

```bash
/root/bench-v2/runner/run-schedule.sh --schedule /root/bench-v2/schedules/core-n3.tsv
```

The schedule uses the preregistered `random.Random(20260711).shuffle(...)` algorithm. No schedule is generated or executed by installation of this candidate. After the full primary schedule, `run-schedule.sh` retries each attempt whose finalized canonical status is `provider_failure` or `runner_failure` exactly once. Refusals, fallbacks, model-session timeouts and incorrect/incomplete model work are preserved as results and are not retried.

## Output normalization

The exact prompt is the frozen task pack's `PROMPT.md`, passed through stdin and independently hashed. The original prompt, copied input, event stream and workspace remain unchanged.

After inference, `normalize_predictions.py` validates `workspace/output/predictions.json` with strict Pydantic models, rejects metric IDs or units outside the frozen task table, enforces release/count bounds, requires `output/calculation_note.md`, verifies every cited `output/` artifact exists, adds the runner `run_id`, and prefixes each `source_artifact` with `<run_id>/workspace/`. It writes evaluator-ready `predictions.jsonl`. Missing or schema-invalid output changes the canonical manifest status to `agent_failure`; refusal and fallback output is never normalized as a scored Fable prediction.

`finalize-run.sh` writes evaluator-compatible `manifest.jsonl`, including canonical status, served models, prompt/pack/environment hashes, trace path and artifact paths. The lower-level `status.json` retains both the raw runner classification and canonical evaluator status.

## Preflight only

Static checks:

```bash
/root/bench-v2/runner/preflight.sh
```

Filesystem, credential, environment, and network-denial probes for both treatments:

```bash
/root/bench-v2/runner/probe-isolation.sh --system codex
/root/bench-v2/runner/probe-isolation.sh --system claude
```

Neutral one-word provider smokes, stored only under `preflight/`:

```bash
/root/bench-v2/runner/smoke-one.sh --system codex --name isolated-ready-v3
/root/bench-v2/runner/smoke-one.sh --system claude --name isolated-heat-flow-v3 --kind engineering
```

The engineering variant is a fixed elementary slab-conduction calculation. After those baseline checks, both systems must also complete a neutral file-operation probe through the byte-identical scored invocation wrapper:

```bash
/root/bench-v2/runner/smoke-one.sh --system codex --name scored-parity-v3-final --kind parity
/root/bench-v2/runner/smoke-one.sh --system claude --name scored-parity-v3-final --kind parity
```

The parity probe exercises the exact model, effort, tool, permission, persistence, user-configuration, MCP, and sandbox flags used for scored work. It independently checks the file produced by the model against a preregistered SHA-256 value and fails unless the event validator reports a real, uncontaminated assistant turn. This proves CLI-command parity; protocol validation, environment comparison, task copying, finalization, and scored time limits are tested separately. None of these smoke commands read a benchmark task.

An earlier v3 candidate passed `--json-schema` to Claude and `--output-schema` to Codex for the otherwise unused final message. The exact parity probe showed that Claude Code 2.1.201 accepted the option but Fable returned a successful plain-text result with no structured object. Both final-message constraints were removed before freeze. The shared scored contract is instead the strict, Pydantic-validated `workspace/output/predictions.json` described above.

Run the integration suite only after creating those v3 smoke artifacts. The classifier regression also uses preserved neutral v1 refusal/fallback event fixtures; it never reads a scored event stream.

```bash
cd /root/bench-v2/runner
/root/.local/bin/uv sync --frozen --dev
/root/.local/bin/uv run pytest
```

## Known model-verification asymmetry

Claude stream events identify the actual assistant model and explicit fallbacks, so the validator can fail closed. Current Codex JSONL identifies the requested CLI session but does not emit an equivalent per-message server model identifier; the runner records the exact model argument, CLI version and event stream, but cannot independently prove the served model from JSONL alone.
